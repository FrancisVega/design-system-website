Iconography uses images and symbols to visually represent an object. They communicate a message, and should be simple, distinct, and informative. Icons should be used sparingly throughout the product to provide clarity and reduce cognitive load on users.

### Background and Internal Padding All icons should be on a transparent background and centered within the artboard boundaries of one of the below standard dimensions. This means that icons could have extra internal horizontal or vertical. **Background: 100% Transparent** **Alignment in container: Centered**
### External Padding Include enough padding around the icons so that no matter the icon size it will always meet the standand touch affordance of 44 pixels or higher. **Target Area: 44 px or higher**
### Color Interactive UI Icons need to pass the same color contrast ratio as typography at a 4.5:1 ratio. UI icons are always a solid color. The color of the icon should depend on the importance of the action of icon. If it is a primary action then the icon should be Blue 30. If the icon is a secondary or supplementary action then icon color should be a neutral, Navy-Gray-8. Blue should always be used with an icon to show that it is interactive. For secondary UI Icons the hover state should be blue to help emphasize its int. **Exception:** There is an exception to the either Blue or Neutral color rule. Icons such as status or notification icons can inherit their parent color. For example, a warning icon is yellow because warning notifications are yellow. **Primary UI Icon Color State** - Normal: Blue 30 #5aaafa - Hover: Blue 20 #7cc - Disabled: @ 50% opacity **Secondary UI Icon Color State** - Normal: Navy-Gray-8 #dfe6eb - Hover: Blue 30 #5aaafa - Disabled: @ 50% opacity
### Weight Icons of the same size should have the same visual weight. One icon should not look heavier or lighter than another icon of the same size. Most Bluemix UI Icons are drawn with a two pixel stroke. Be careful when resizing icons that the 2px stroke does not increase or decrease causing icons to look uneven. When scaling icons you should always start each icon at the same base size (preferrably the default of 32px). **Glyphs** or 16 pixel icons should always be a filled icon. This adds visual weight to the icon allowing it to maintain its proper emphasis and stay legible. Fine stroke weights can disappear or break at the glyph size.

### Pixel Grid When drawing or re-sizing icons make sure the vector always aligns to the base pixel grid. This insures pixel clarity and crispness on all screen ratios.
