Iconography uses images and symbols to visually represent an object. They communicate a message, and should be simple, distinct, and informative. Icons should be used sparingly throughout the product—only to provide clarity and reduce cognitive load on users.

Adding Icons

If you'd like to add an icon to our library, submit a Pull request in our GitHub Repo.

If you're not comfortable with making a Pull Request, please submit an issue in the repo with the icon attached.

  • Fork the GitHub repo
  • Clone it to your local system
  • Add icons in the proper folder (you may add your own) within the svg/ folder
  • In your terminal run npm install
  • Run npm run build to rebuild the sprite.svg
  • Submit a pull request and rejoice!

Styling

In this usage, you style the SVG in CSS. Make sure to give it a width and height (default is 100% of parent). Some icons are colored while others are black by default. You can edit the fill to change this. See the reference page for a guide with what xlink:href to use.
