Guidelines

User Experience

Don't overwhelm the user

A user should never be faced with too many choices or too much information without any obvious next steps. A developer’s experience with Bluemix should be stable in that he/she does not experience a lot of guidance and then none at all. Utilize progressive disclosure and the Gestalt principles where appropriate.

Make the 80% use-case great and make edge cases possible. It is easy to get bogged down making sure obscure edge cases don’t break a design, but in order to keep Bluemix simple and usable, you should optimize for the 80% use case and simply ensure edge cases are possible.

It's never simple enough

Enterprise software is complex and as a result it has a reputation for being horribly unusable. As a designer on Bluemix you must assume your design is never simple enough.

Create designs that can grow with the platform

Bluemix is a rapidly growing platform that is constantly expanding in scope and capability. Designs need to be resilient and able to adapt as the platform grows in complexity and capability.

Transparency

Let the user know what’s going on in the background and allow them to peek under the hood a little. If we are giving the user something by default they didn’t explicitly ask for, tell them why.

The “Hello World” approach

Many developers like to start from the basics and figure out how something works by playing with it, not by reading about it. Every developer started their development career with a simple “Hello World” application. Bluemix should leverage these habits and allow developers to start from a state of “Hello World” and then grow their own understanding through experimentation.

See the IBM Design Interaction Guidelines for more guidance.
