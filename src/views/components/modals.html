General

  • Modals should not contain scrolling content
  • If your modal contains informational content only, consider using a Notification instead
>

Dismissal

Modals can be dismissed in 3 ways:

  • Using the “x” in the upper right hand corner of the modal
  • Pressing the ESC key
  • Clicking / touching outside of the modal area

Structure

Header

Include a heading within the modal that mirrors the action or button that was clicked by the user. Headers include a close button (“x”) in the upper right hand corner of the modal.

Body

The body content within a modal should be as minimal as possible. Some base elements that can be used in modals are: form fields, text area, select, and radio buttons.

Footer

The footer area of a modal typically contains a set of buttons. Refer to button guidelines for usage.

Basics

There are two different types of modals, passive and transactional. Passive modals contain information while transactional modals contian information and have actionable buttons in the bottom left hand corner. Refer to the button style guidelines for styling. On mobile the button/action bar is sticky to the bottom of the screen. Both passive and transactional modals can have a label above the title, however it is optional.

Typography

Modal Labels are optional and should be set in all caps when used. Modal Titles should be set in title case with the first letter of each word capitalized. Modal Titles should not exceed more than 3 words. Modal body copy text should be sentence case with only the first word of the sentence capitalized. Important directions or information in the body copy can be bold, othewise it should be in roman. Modal labels, titles, and text should all follow the Bluemix Typography font stack for sans-serif.
