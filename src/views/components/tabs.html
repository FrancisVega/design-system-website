Text

For the Service Details Page, Overview, Getting Started, and Connected Objects are required across all services. If custom UI is needed for your service, there are two options.

Required Tabs

Each text title for a tab should appropriately describe its content and set user expectations. Titles should be as concise as possible: one, two, or if necessary, three words. Keep in mind that at mobile widths the character length of the title will impact the experience. Icons are not permitted in the tab titles.

  1. Manage Tab
    If you have minimal need for custom UI for your service, it can be contained within the Manage Tab.
  2. Custom Tabs
    If your service has more advanced taxonomical needs, additional custom tabs may be included.

Number of Tabs

  • There is a maximum of six tabs displayed. This is to maintain an uncluttered UI and reduce cognitive load for users.
  • If you require additional tabs, please see the Overflow Menu component for further guidance.
  • Tabs should be ordered from left to right by frequency or expected frequency of use by the user. Alternatively, if each tab is used equally, tabs should be placed in the order in which a user flows through each area of content.
  • Tabs with related content should be grouped adjacent to each other.

Hierarchy

Only one level of tabs is used. This is to ensure a consistent, simple, and easily navigable experience for users. An intuitive taxonomy and descriptive tab titles can go a long way toward a good user experience.

Exceptions

Secondary Level

A secondary level of tabs is permitted in the case of displaying multiple sets or views of the same content, or switching between discrete, but related content. Therefore, the secondary level should be considered a Content Switcher rather than navigation.
The second level manipulates the view or the content within its parent tab, and thus is part of the active tab. It should look and feel like as such. Please see Content Switcher for more detail.
