{{#markdown}} ## Structure ### Header Row Headers guide users to the data that is presented below. Use meaningful text to label the table data and provide clarity to the content below. ### Number of Columns For large sets of data, it is preferred to use rows rather than columns, to guide the eye across the page. Readers can scan quickly down a single row, but find it harder to scan across many columns to relate connected data. ### Zebra Stripes Alternating colors on rows increases readability, and helps the guide the users through the data in the table. ## Interactions ### Search Search functionality within a table should follow the convention of a small search. ### Sorting Usability of your table will be greatly improved by making data sortable. By making header labels clickable, it allows the user to sort the table data up/down to their liking. Whichever attrribute is being sorted should be labeled with a caret. ### Batch Actions Batch actions are functions that can be performed on multiple items within a table. Use an overflow menu to present batch actions to the user. The overflow menu appears when the user clicks on the icon. If there is only one batch action available for the table data, that action should be styled as a primary button. ### Inline Actions Inline actions are functions that can be performed on a specific table row. When the user hovers over a table cell, an overflow menu icon appears which contains actions related specifically to that table row. ### Expandable Rows Expandable rows allow for minimal information to be shown at a high level, and a more detailed description shown nested within the row. ## Additional Content Depending on the type of data within a table, any of the below options can be used to load additional content. It is important to note that only 1 of these solutions should be implemented. ### Pagination Pagination divides table data into separate pages. Pagination should be accompanied by an option that allows the user to change the number of items per page. ### Lazy Load Lazy loading loads additional content as the user scrolls in their viewport. ## Example
