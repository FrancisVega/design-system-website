{{#markdown}} ## General ### Text Each tab title describes its content and sets user expectations. Titles are concise and use one to two words maximum. Keep in mind that at mobile widths, the character length of a title will impact the experience. Icons are not permitted in the tab titles. ### Number of Tabs A maximum of six tabs may be displayed. This is to maintain an uncluttered UI and reduce cognitive load for users. ### Order Tab order should be consistent across service instances. Tabs with related content should be grouped adjacent to each other. ## Standard Use Cases Tabs are used within the Service Dashboard. ### Manage The Manage tab contains learning content and additional information that aids in productive use. ### Service Credentials The service credentials tab shows the service instance credentials to an end user. Currently, users have the ability to create mulitple sets of credentials. Services that do not use credentials will not be required to have this tab. ### Connections This tab provides a way for users to bind their service instance to an existing app or service instance. Service to service bindings will be supported in the near future. ### Getting Started Get Started is defined as the bare-minimum set of usage instructions an end-user needs to get up and running. Getting started content may manifest itself in different ways across the platform. Instill user confidence by presenting the quickest path to success. **Use Case** Get Started at the instance level provides content to help a user to get up and running as fast as possible. At this level, getting started content manifests itself in the Instance Dashboard under the Getting Started tab. Upon creation of a service instance, the content on the Getting Started tab is the first thing users encounter and will likely dictate how successful they are with the service. **Implementation** Service providers should have consistent getting started documentation using the Bluemix Getting Started Template and Guidelines. Content on the Getting Started tab is populated directly from Bluemix Documentation once service providers register their service. **Example**
