### Basics Forms are made up of several different base elements and components. The order in which these elements are arranged is flexible, but should always follow the padding guidelines below. Forms can be one column or two. The width of each column varies based on the content and layout of the design. On mobile, forms can only have one column. **Form width: Varies based on content and layout**
Please refer to the text field, dropdown, checkbox, radio button, toggle and button pages for additional styling and usage details. All icons can be found in the Bluemix Icon Repo or the Iconography Library page.

### States There are two different validation states for forms. One is when a required field is left incomplete by the user. This prompts an in line error notification and underlines the effected fields in red. Please refer to the notifications page for usage and styling. The second state occurs when a field is not filled out to meet the basic requirements for that particular field. For example, a password with 16 characters is required, but a password with only six characters is inputted. This prompts an error notification, and underlines the erroneous fields in red.

### Typography
Form title's should be set in title case, with the first letter of every major word capitalized. Form text should follow the Typography font stack for san-serif. Please refer to the text field, dropdown, checkbox, radio button, toggle, and button pages for specific typography details on the components.
