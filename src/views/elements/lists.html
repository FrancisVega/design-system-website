Length

Generally, lists should be used when there is a single line item of information per line. For larger or more complex sets of data, consider using a Table.

Order

The default order of a list should make sense given the context, and help facilitate the user's goal. For example, if the list is about resource use, perhaps the default order should be highest resource use to lowest, instead of being sorted alphabetically.

If there is no such meaningful default, alpabetical or numeric order should be used. Grouping items by categories into smaller, more specific lists may be more meaningful to users in some contexts.

Filter & Sort

Lists can be filterable and sortable, allowing the user to manipulate and view the content as they see fit.

Structure

Basics

There are two types of lists: ordered and unordered. Level 1 ordered list points are marked numerical with Level 2 points marked with a dash. Level 1 unordered list points are bulleted with Level 2 points marked with a dash.

Padding

Lists cannot have any element or text within 16 px / 1em of its borders.

Typography

Level 1 text in a list should be set in bold/700 and title case, with the first letter of every word capitalized. Level 2 text in a list should be set in roman/400 and title case as well. List text should follow the Bluemix Typography font stack for sans-serif.
