Labels

Labels are essential to usabililty of text fields. They provide guidance to the user in reference to what data they should input. Do not place a label inside a text field. If further clarification is needed, use placeholder text.

Placeholder text

Placeholder text is used to complement a field and provide additional assistance to the user. It is helpful to provide examples of the data format required. Placeholder text should dissapear once the user has inputted data into the text field.

Validation

Real time validation helps to streamline the process and keep data clean when the user is filling out data. Including helper or error text before, during, or after a user interacts with a text field can aid the user in making correct decisions.

Mobile

Text fields on mobile should typically be stacked vertically, with labels and or hints above each field. As mobile text fields increase in size (especially length), the limitation on space becomes less marked.

Structure

Basics

Text fields are single color filled at 10% opacity. The width of a text field should appropriately fit the design and layout of the content.

Select Box Background Color: rgb (255,255,255) at 10% opacity

Text Field Basics | units in px / em

States

  • Default Non-Selected State: Browser Defined
  • Hover: 1px blue outline (rgb 90, 170, 250)
  • Form Validation: 2px bottom border (rgb 255, 80, 80)
  • Help: Tool tip should be left aligned with the text field box and underlined to indicate a link
  • Disabled: 50% opacity

Hover Border weight: 1px
Hover Border color: rgb (90, 170, 250)
Form Validation Bottom Border weight: 2px
Form Validation Bottom Border color: rgb (255, 80, 80)

Text Field States

Groupings

Text fields can be grouped horizontally and vertically for tablet and desktop views. Text fields cannot be grouped horizontally for mobile views.

Padding between vertical groupings: 32px / 2em
Padding between horizontal groupings: 16px / 2em

Text Field Grouping Orientations

Typography

Text field labels should be set in title case with the first letter of every word capitalized and should not exceed more than 3 words. Text field text should be set in sentence case with only the first letter of the first word capitalized. Text fields text should follow the Bluemix Typography font stack for sans-serif.
