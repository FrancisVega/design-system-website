Usage

Toggles should be used for binary actions that occur immediately after the user "flips" the toggle switch. They are commonlly used for "On/Off" situations.

Titles

Titles can be used to accompany Toggles to provide further clarification as to what action the toggle will perform.

Labels

Use a label to accompany a toggle so the user is aware of the action that will occur. Labels should be no more than 4 characters long, and appear on both sides of the toggle.

Language

Labels should be adjectives, which describe the state of the object affected, rather than a verb. Verbs describe an action, in which case the user does not know whether the action has taken place or is yet to take place.

Structure

Basics

Toggles have two states: action on and action off. When an action state is on, the toggle button should be solid and on the right end of the toggle bar.

When an action state is off, the button should have a 2px border with no fill and be on the left end of the toggle bar.

The font for the unactive state should be lighter than the active state.

Toggle Button height & width: 24px / 2.5em
Button Border: 2 px / .125 em
Font size: 14px / .875 em
Font weight: normal / 400

Toggle | units in px / em

Padding

Toggles can not have any element or text within 16 pixels / 1em of its borders.

External Padding: 16px

Toggle Padding | units in px / em

Typography

Toggle text should be set in title case with the first letter of each word capitalized. Toggle labels should not exceed more than 3 words. Toggle text should follow the Bluemix Typography font stack for sans-serif.
