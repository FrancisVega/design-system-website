Labels

Effective form labeling helps users understand the required information to put into the field.

As a space-saving method, often placeholder/hint text is used as a label. This is not best practice, since the interaction of the label disappearing can cause issues for people with cognitive issues or who use a screen magnifier. It also increases cognitive load for all users who would have to remember the purpose of the field after they filled it in with text.

  • Label must be visible when the field gets focus
  • The label must be announced to the screen reader on focus
  • Ensure helper text that appears under a field is read when the assistive technology user stops at the field using aria

Validation

Real time validation helps to streamline the process and keep data clean when the user is filling out data. Including helper or error text before, during, or after a user interacts with a text field can aid the user in making correct decisions.

Order

The order of the list should be based on the frequency of use. If applicable, the list should be in increasing order relative to the content. In cases of forms, alternative orders such as alphabetical may be more fitting. A horizontal rule can be used to group similar items together.

Accessibility

The default select element is completely usable either with a mouse or a keyboard, following this process:

  1. Mouse: move your cursor over the select element
    Keyboard: use the tab key to focus the select element
  2. Mouse: click on the select element
    Keyboard: press enter
  3. Mouse: move your cursor over the desired option
    Keyboard: use the top and bottom arrow keys to pick an option
  4. Mouse: click on the desired option
    Keyboard: press enter

While you can make a CSS dropdown easily usable by a mouse, making it usable with keyboard navigation is complex.

Structure

Basics

Select boxes are single color filled at 10% opacity. The width of a select dropdown box should appropriately fit the design and layout of the content.

Select Box Background Color: rgba(255,255,255) at 10% opacity
Select box height: 40px / 2.5em

Padding

Space between label and select box: 10px / .625em
Internal side padding: 16px / 1em

  • Hover: 1px blue outline (rgb 90, 170, 250)
  • Open: Style determined by browser

  • Help: Tool tip should be left aligned with the text field box and underlined to indicate a link. Disabled: 50% opacity

Groupings

Select dropdowns can be grouped horizontally and vertically for tablet and desktop views. Select dropdowns cannot be grouped horizontally for mobile views.

Padding between vertical groupings: 32px / 2em
Padding between horizontal groupings: 16px / 2em

Typography

Select dropdown labels should be set in title case with the first letter of every word capitalized and should not exceed more than 3 words. Select dropdown text should be set in sentence case with only the first letter of the first word capitalized. Select dropdown text should follow the Bluemix Typography font stack for sans-serif.

Select Label font-size: 14px / .875rem
Select Label font-weight: bold / 700
