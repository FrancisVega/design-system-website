General

Buttons are used primarily on “action items.” Some examples are “Add, Save, Delete, Sign Up.” Do not use buttons as navigational elements; use links instead. A link takes the user to a new page and is not associated with an action.

Text

In the UI, buttons and descriptions of buttons use the verb of the actionable step. For example, “Delete” instead of “OK,” “Yes,” or “Done.”Â  “Save” can be used if the action is to finalize the editing of existing information.

Icon Usage

  • Use small icons (16px or less) within buttons.
  • Small icons are distinguished by their solid shape and knocked-out details.
  • Icons should always appear to the right of the text.
  • Icons used in buttons must be directly related to the action that the user is taking.
  • Icons must be the same color value as the text within a button.

Structure

Basics

Primary buttons are single-colored, filled buttons with a contrasting color as the text label.

Secondary buttons have a solid 2px border with no fill. The text and border color on secondary buttons styles should be the same color.


Button height: 40px / 2.5em
Padding left and right of text: 16px / 1em
Font size: 14px / .875 rem
Font weight: bold / 700

Padding

A button can not have any element or text within 16 pixels / 1 em of its borders.

External Padding: 16px / 1em

Groupings

In a set of buttons, the primary button style is positioned on the outside of the set. The secondary button is positioned on the inside of the set.

Space between buttons: 16px

Button Groupings | units in px / ems

Button with Icons

There must be 16 pixels / 1 em between the text and the icon within the button.

Button with Icon | units in px / ems

States

Primary Style

Focus: Browser defined

Hover: Filled and one shade lighter on the IBM Design Language color scale than normal color. Text stays the same color.

Disabled: 50% opacity

Secondary Style

Focus: Browser defined

Hover: Filled with the same color as the stroke in the normal state. Text should change to a contrast color.

Disabled: 50% opacity

Typography

Button text should be set in title case, with the first letter of each word capitalized. Button labels should not exceed more than 3 words. Button text should follow the Bluemix Typography font stack for sans-serif.
