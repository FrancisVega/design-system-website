Guidelines

Length

Generally, lists should be used when there is a single line item of information per line. For larger or more complex sets of data, consider using a [table](#). Both lists and table may alternatively be displayed as a grid.


Scrolling

Lists should only scroll vertically. If more horizontal space is needed, consider using a larger line height per item or adjusting to a grid view to accomodate the additional information.


Order

The default order of a list should make sense given the context and facilitate the user's goal. For example, if the list is about resource use, perhaps the default order should be highest resource use to lowest and not alphabetically.

If there is no such meaningful default, alpabetical or numeric order should be used. Grouping items by categories into smaller, more specific lists may be more meaningful to users in some contexts.


Filter & Sort

Even with a meaningful default order, in many cases, lists should be filterable and sortable, allowing the user to manipulate and view the content as they see fit.
