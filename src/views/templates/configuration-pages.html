Configuration pages appear when a user has clicked on a category from the Bluemix catalog. The user is typically creating a new instance of a service, Container, Virtual Machine, or CF App.

{{#markdown}} You can find the source file for these templates in the Bluemix Design Kit. ## Service configuration Service providers provide meta-data within the Bluemix Service Operation Plan (SOP) that will auto-populate into their Service Configuration page. The meta-data fields required are: * Name (XYZ Service) * Long description * Link to Docs * Author (IBM, Third Party) * Publish Date * Type (Service) * Location * Feature description (bullet points) * Screenshots or images of service UI * Pricing plans
