Instance Dashboards are used for showing information and details of an instance that exists in the user's dashboard. They appear when a user clicks on a specific item in their dashboard. Typically, these instances can include CF Apps, Services, Virtual Machines, and Containers. Use these templates as a starting point and foundation for your design.

You can find the source file for these templates in the Bluemix Design Kit.

Service Dashboard

Service dashboards may be accessed from the Global Dashboard, or after a user provisions a service from the Bluemix catalog. Service dashboards serve as locations for users to learn more about services, configure their service instance and connect it to an existing app or service instance.

As shown in the example above, Service Dashboards are comprised of a series of tabs that are defined in the service broker meta-data.

Manage Tab

Content aiding in a user's productive use workflow, and any additional learning content should surface in the Manage tab. Consider providing pertinent introductory material such as links to service-specific API documentation and demos so users quickly and effectively get up and running.

Getting Started Tab

Get Started at the instance level should be all about getting a user up and running as fast as possible.

Upon creation of a service instance the content on the Getting Started tab is the first thing a user encounters and will likely dictate how successful they are with the service.


Implementation
Service providers should provide consistent getting started documentation using the Bluemix Getting Started Template and Guidelines. Content on the Getting Started tab is populated directly from Bluemix Documentation when registering your service.

Service Credentials Tab

The Service Credentials tab shows the service instance credentials to an end user. Currently, users have the ability to create mulitple sets of credentials. Services that do not use credentials are not required to have this tab.

Connections Tab

This tab provides a way for users to bind their service instance to an existing app or service instance. Service to service bindings will be supported in the near future.

Custom Tab

We understand some services will require an additional level of functionality not currently supported with the default tabs. Service providers who validate a need for an additional tab are permited one custom tab.
